## 流程表（給客戶看）

| 大約時間 | 做什麼 | 怎樣算通過 |
|----------|--------|------------|
| 0:00 | **開場說明** | 了解今天測什麼、已知限制（見開測信第四節） |
| 0:15 | **系統設定** | 各項政策畫面看得見、改完能儲存、沒有一直報錯 |
| 0:45 | **匯入週更表** | 檔案上傳成功；職位為 PT／OT 等代碼；員工姓名與系統內一致 |
| 1:00 | **啟動智能排班** | 出現「員工工作表」預覽：誰、哪個時段、哪些院友 |
| 1:15 | **檢查衝突提示** | 若有紅字／警示，內容看得懂、符合院舍理解 |
| 1:30 | **儲存排班** | 按儲存成功；重新整理後資料仍在 |
| 1:45 | **（選測）換前線帳號** | 前線同事只能看該看的，不能代主管儲存全院排班 |
| 2:00 | **記錄問題與簽名** | 重大問題已登記；驗收紀錄簽名 |

## 常見狀況（不用緊張）

| 您可能看到 | 原因／處理 |
|------------|------------|
| 更表說「職位須為 PT…」 | 更表「職位」欄請填 **PT、OT** 等英文代碼，勿填中文職稱 |
| 更表說「找不到員工」 | 更表姓名須與「員工管理」內**完全一致**（含職位） |
| 按儲存沒反應 | 請用**主管／Team Lead**帳號；先確認畫面顯示「無衝突」 |
| 院友名單是空的 | 需用**主管**帳號；一般前線帳號可能看不到院友主檔 |
| 儲存後員工 App 看不到班表 | **正常**（試用版第一期）；目前僅在主管畫面預覽與儲存紀錄 |
