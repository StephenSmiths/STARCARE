## 二、UAT 劇本（對齊 PDF【3】五步）

**建議總時長**：約 45～60 分鐘。通過標準：下列 **S1～S10** 預期皆符合，且無 **P0** 缺陷。

| 編號 | 操作 | 預期結果 | 通過 |
|------|------|----------|:----:|
| **S1** | 以 **Team Lead** 登入，側欄進入 **智能排班**（`#scheduling`） | 頁面載入；無持續「無法連線載入」錯誤 | ☐ |
| **S2** | 進 **系統設定**，檢視 **智能排班設定**／**復康服務基本設定** | 可見午休、非治療時段、Pass 次序等；可本機儲存或（有雲端時）提交政策版本 | ☐ |
| **S3** | 回排班頁，**週更表匯入**：上傳 CSV → **預檢** | 預檢通過或錯誤列清楚標示行號／原因 | ☐ |
| **S4** | 確認更表後勾選「已確認週更表」（若有） | 可進入後續排班步驟 | ☐ |
| **S5** | 點 **啟動智能排班**（或同等「開始排班」） | 產生指派預覽；可見 Pass／覆蓋院友／KPI 或統計卡更新 | ☐ |
| **S6** | 檢視 **衝突／警示** 區（若有） | 衝突說明可理解；週三資助警示（若適用）與業務一致 | ☐ |
| **S7** | 檢視 **資助復康** 與 **認知** 兩軌（若畫面分區） | 認知軌不混入資助復康 KPI；名單／統計分示 | ☐ |
| **S8** | **一鍵儲存排班結果** | 成功提示；無重複提交造成異常 | ☐ |
| **S9** | 重新整理頁面或至 **儀表盤** 看審計／摘要 | 可見排班或儲存相關審計紀錄（依產品設定） | ☐ |
| **S10** | 以 **Staff** 登入再進排班（選測） | 權限符合角色：可檢視範圍內操作；不應出現越權儲存全院結果 | ☐ |

## 四、已知限制（測試時請知悉）

1. **排班演算**目前主要於**瀏覽器端**執行，結果仍依 PDF【3】Pass 與間隔規則；與「全後端 MUST」工程描述有架構差異。  
2. **週更表活動**：依 **職位＋服務類型＋工作計劃目錄** 擇一；尚未依「每位院友個別復康計劃」優先匹配。  
3. **私位**週目標程式預設為 **1 次／週**；若貴院裁定「每週最多 2」須另案確認。  
4. **Demo 模式**（無 Supabase 連線）僅供畫面演示，**不可**作為正式驗收。
